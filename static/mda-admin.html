Household Data

Upload the household file (.xlsx or .csv). For .xlsx: must contain a Forms sheet and optionally a Repeat-group_indv sheet. For .csv: treated as the Forms sheet only, and individuals stay at their current values (use .xlsx if you also need to refresh individuals).

Upload the baseline enumeration file (.xlsx or .csv) with State, LGA, Ward, Settlement, Total Treated columns. The R5 Target multi-sheet format requires .xlsx; CSV uploads use the simpler single-sheet pivot.

State Boundary Shapefiles

Upload one ZIP containing all four boundary levels (LGA, Ward, Settlement, Grid). Each layer is recognised case-insensitively by its .shp filename (lga, ward, settlement, grid / gridded), with sidecars (.shx, .dbf, .prj, optional .cpg) sharing the same basename. Boundaries are shared across all rounds in the state.

Click to select state boundaries ZIP
Accepts .zip • LGA + Ward + Settlement + Grid in one file

Why it takes time: the server reads every polygon in your ZIP, re-projects it to the dashboard's coordinate system (EPSG:4326), and writes one row per feature into PostgreSQL. Grid is by far the largest layer (≈600k 30×30 m cells for a state) so it dominates the wall-time — LGA and Ward finish almost instantly, Settlement takes a minute or two, and Grid runs in the background for several more minutes. You can leave this tab open or come back later — the import continues server-side and won't restart.
Recompute Geographic Analytics

Re-runs spatial QC and rebuilds the settlement completeness rollup for the selected project. Run this after uploading or re-uploading boundaries when forms are already in the database — otherwise the Geographic View will show 0% completeness and household forms will stay flagged as “GPS outside LGA” from a stale state. Idempotent and safe to re-run; takes 10–60 seconds depending on grid size.
